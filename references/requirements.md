# 软著规则摘要

本文件用于给使用该 skill 的代理提供软著申请和审查背景，不用于替代正式申报系统说明。

## 核心材料

根据《计算机软件著作权登记办法》，软件著作权登记通常涉及三类核心材料：

- 软件著作权登记申请表
- 鉴别材料
- 相关证明文件

鉴别材料通常包括：

- 源程序
- 文档

常见文档形态包括：

- 用户手册
- 操作手册
- 设计说明书

## 常见审查点

代理在生成材料时，应优先避免这些问题：

- 软件名称在申请表、说明书、代码文档中不一致
- 主要功能、开发目的、技术特点之间前后矛盾
- 文档主题与软件名称不匹配
- 代码文档里残留旧项目名称、旧包名、旧实体名
- 文档过短或结构过散，无法支撑系统完整性
- 内容明显套壳、与系统简介无关、复用痕迹过重

## 鉴别材料常见口径

《计算机软件著作权登记办法》规定了源程序和文档的基本提交口径。实务中常见理解是：

- 源程序和文档通常按前、后各连续 30 页准备
- 总页数不足 60 页时，可以全部提交
- 申请表、说明书、代码文档中的软件名称应保持一致

本 skill 参考这些口径生成固定格式的材料，但不负责在线申报和主体证明文件。

## 补正与不予登记风险

在规则和常见办理说明中，以下情况容易触发补正或影响登记：

- 申请材料不齐全或填写不规范
- 申请人信息、权属依据、名称或版本信息不一致
- 提交材料存在明显权属争议
- 软件名称或内容违反相关规定
- 鉴别材料无法识别软件独创表达

## 本 Skill 的定位

本 skill 的定位是：

- 快速生成三件套草案
- 保持模板风格统一
- 降低手工改写工作量

本 skill 不处理：

- 在线正式填报
- 主体身份证明
- 委托书或合作开发协议
- 法律真实性核验
